### 1. Waterfall Model / 瀑布模型

**What it is / 是什么:**
The Waterfall model is a step-by-step method. You finish one phase (like design) completely before moving to the next phase (like coding). It is simple and easy to follow, but going back to fix things is very hard.
瀑布模型是一种按顺序开发的方法，完成一个阶段（如设计）之后才能进入下一个阶段（如编程）。它简单易懂，但很难回头修改。

**When to use it / 适用场景:**
Use it when the requirements are very clear and will not change. Good for short, simple projects with a fixed plan.
用在需求非常明确且不会变的项目。适合有固定计划的短期简单项目。

**Real-world example / 真实案例:**
NASA used Waterfall for space shuttle software because safety is the top priority and requirements must be fixed early.
NASA 在航天飞机软件中使用瀑布模型，因为安全第一，需求必须早早确定。

---

### 2. Agile Methodology / 敏捷方法

**What it is / 是什么:**
Agile is not a fixed set of rules, but a set of ideas. It focuses on making small, working pieces of software quickly, then getting feedback from the customer and making changes. It welcomes change at any time.
敏捷不是一套固定规则，而是一套思想。它注重快速做出能跑的小块软件，从客户那获取反馈，然后修改。它随时欢迎变化。

**When to use it / 适用场景:**
Use it when you don‘t know all the requirements from the start. Great for startups or projects where user needs change often.
用在你一开始不清楚所有需求的时候。适合初创公司或用户需求经常变的项目。

**Real-world example / 真实案例:**
Spotify uses Agile. It has many small independent teams that can release new music features quickly and get user feedback.
Spotify 使用敏捷方法。它有很多独立的小团队，能快速发布新音乐功能并获取用户反馈。

---

### 3. Scrum Framework / Scrum 框架

**What it is / 是什么:**
Scrum is a specific way to do Agile. The team works in short, fixed periods called “Sprints” (usually 2 weeks). Every day, the team has a quick stand-up meeting. Scrum has clear roles: Product Owner, Scrum Master, and Development Team.
Scrum 是做敏捷的一种具体方式。团队在叫“冲刺”的短周期（通常 2 周）内工作，每天快速开会。它有清晰的角色：产品负责人、Scrum Master 和开发团队。

**When to use it / 适用场景:**
Use it for complex projects that need strong teamwork and regular customer feedback. It helps when the product needs to grow step by step.
用在需要紧密团队合作和定期客户反馈的复杂项目。适合产品需要一步步成长的情况。

**Real-world example / 真实案例:**
Microsoft uses Scrum in parts of the Azure DevOps and Office 365 teams to deliver updates faster and respond to user feedback.
微软在 Azure DevOps 和 Office 365 的部分团队中使用 Scrum，以更快交付更新并响应用户反馈。

### 4. Kanban / 看板法

**What it is / 是什么:**
Kanban uses a visual board with columns like “To Do,” “Doing,” and “Done.” The main goal is to see the work flow and limit how many tasks are in progress at one time. There are no fixed time-boxed iterations.
看板使用可视化面板，有“待办”“进行中”“已完成”等列。主要目标是看清工作流程，并限制同时进行的任务数量。它没有固定的时间周期。

**When to use it / 适用场景:**
Use it when you have a continuous flow of work, like fixing bugs or handling support tickets. It is very flexible for changing priorities.
用在有持续工作流的时候，比如修 Bug 或处理支持请求。它在优先级变化时非常灵活。

**Real-world example / 真实案例:**
Toyota first created Kanban for car making. In software, teams like the BBC iPlayer team use it to manage ongoing bug fixes and small features.
丰田最早在看板用于汽车制造。在软件中，像 BBC iPlayer 的团队用它来管理持续的 Bug 修复和小功能。

---

### 5. DevOps

**What it is / 是什么:**
DevOps is a culture and a set of practices. It brings together the Development (Dev) team and the Operations (Ops) team so they work together, not separately. It uses automation tools to build, test, and release software very fast and safely.
DevOps 是一种文化和实践集合。它把开发团队和运维团队合在一起，让他们一起工作而不是各干各的。它用自动化工具快速、安全地构建、测试和发布软件。

**When to use it / 适用场景:**
Use it when you need to release software updates many times a day or week. It is great for cloud-based applications that must always be online.
用在需要一天或一周内发布很多次更新的情况。适合必须一直在线、基于云的应用。

**Real-world example / 真实案例:**
Amazon uses DevOps. It can deploy code every few seconds using automated pipelines, allowing fast updates to its global websites.
亚马逊使用 DevOps。它用自动化流水线可以几秒钟就部署一次代码，快速更新全球网站。

### 6. Spiral Model / 螺旋模型

**What it is / 是什么:**
The Spiral Model combines Waterfall’s step-by-step approach with making prototypes (trial versions). Its most important feature is “risk analysis.” In each cycle (spiral), the team first checks what could go wrong before moving forward.
螺旋模型结合了瀑布的一步步方式，加上做原型（试用版本）。它最大的特点是“风险分析”。在每个循环（螺旋）里，团队先看看有什么可能出错，再继续往前走。

**When to use it / 适用场景:**
Use it for very large, high-risk projects where failure could cost a lot of money or be dangerous. It is good when you need to manage serious risks.
用在规模大、风险高、失败代价很大或很危险的项目。适合需要严格管理风险的情况。

**Real-world example / 真实案例:**
The military uses the Spiral Model for complex defense software (like missile systems), because managing risk is the most important thing.
军方在复杂的国防软件（如导弹系统）中使用螺旋模型，因为管理风险是第一要务。
